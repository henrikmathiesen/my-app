Some Pipes

Angular has built in pipes. They are case sensitive.

Pipes are pure by default

They only run when a value type is changed or a reference is changed. But not when mutating state. Pure and impure pipes

Parameterizing a pipe

A pipe can accept any number of optional parameters. To add parameters to a pipe, follow the pipe name with a colon and then the parameter value (such as currency:'EUR'). If the pipe accepts multiple parameters, separate the values with colons (such as slice:1:5)

Using pipe (built in and custom) from code behind

See some-pipes.component.ts, need to provide them (app.module.ts)

json Pipe Example

Useful for debugging

Without pipe

{{ obj }}

With pipe (also chaining uppercase for fun)

{{ obj | json | uppercase }}

Date Pipe Example

Without pipe

{{ vacation }}

With pipe

{{ vacation | date : 'yyyy-MM-dd, h:mm' }}

Just testing a Date Object

{{ someDate }}

{{ someDate | date: 'yyyy-MM-dd, h:mm' }}

Custom Pipe (need to declare it, see app.module.ts)

{{ 'Lorem ipsum dolores es sitamet' | truncateWords : 3 }}

Testing edge case: {{ '' | truncateWords : 0 }}
